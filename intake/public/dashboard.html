§ VII

SharePoint activation.

your move

An app registration exists in the CRREM tenant for Mail scope. You're now a tenant admin. Three actions, three Azure clicks. Once done, Clem reads CRREM SharePoint in real time.

STEP 1 · ADD SHAREPOINT PERMISSION~60 SEC

Click the button below — it opens the BRIDGE Graph app in Azure Portal for the CRREM tenant directly.

Open the app's API permissions →
  1. Click + Add a permission → Microsoft Graph → Application permissions.
  2. Search Sites.Read.All → check the box → Add permissions.
  3. Click Grant admin consent for CRREM (blue button above the list). Confirm.

You'll see green checkmarks next to all permissions when consent is granted.

STEP 2 · MINT A CLIENT SECRET~30 SEC
Open the app's Certificates & secrets →
  1. Under Client secrets, click + New client secret.
  2. Description: bridge-intake-sharepoint. Expires: 24 months.
  3. Click Add.
  4. Copy the secret value immediately — Azure only shows it once.
STEP 3 · HAND IT BACK~5 SEC

Drop the secret value in our chat. I wire it into the cloud env vars, redeploy, and SharePoint goes live.

Tenant ID and client ID are already known on my side — I just need the new secret.
